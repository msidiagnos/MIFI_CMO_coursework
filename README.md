# Аналитический отчёт: прогнозирование эффективности химических соединений

## 1. Введение

Цель работы — на основании предоставленных данных от химиков необходимо построить прогноз, позволяющий подобрать наиболее эффективное сочетание параметров для создания лекарственных препаратов

## 2. EDA

Исходный набор данных: 1001 строка, 214 столбцов. После удаления столбца id и добавления логарифмических целевых признаков осталось 216 столбцов. После удаления строк с пропусками и дублями осталось 966 строк.

### 2.1. Распределение целевых переменных

Гистограммы исходных целевых переменных показывают ненормальное распределение (правый хвост). Логарифмическое преобразование (log1p) значительно улучшает нормальность.

Наличие выбросов в целевых признаках (и в остальных тоже) подтверждается методом IQR: IC50 - 14.49%, SI - 12.3%, CC50 - 3,62%.

### 2.2. Корреляционный анализ

Корреляции между признаками и целевыми переменными слабые (около трети оставшихся после FE признаков имеют крайне слабые корреляции). Есть сильные корреляции между парами признаков, поэтому при предобработке из каждой группы оставлен только один. Есть признаки с нулевой дисперсией (константные).

### 2.3. Выбросы в признаках

Анализ показал, что часть признаков содержат до 25% выбросов. Поскольку выбросы могут нести важную химическую информацию, удаление не производилось. Применение RobustScaler не дало практически никакого прироста в качестве моделей. Принимая решение об удалении\не удалении выбросов из признаков или целевых переменных опирался на открытые источники, например, https://stats.stackexchange.com/questions/200534/is-it-ok-to-remove-outliers-from-data

В целом, мнение в различных статьях сводится к тому, что удалять выбросы не всегда целесообразно.

## 3. Предобработка данных

- Удалены дубликаты.
- Удалены константные признаки.
- Удалены высококоррелированные признаки (порог 0.90).
- Целевые переменные для регрессии логарифмированы (добавлены новые признаки logIC50, logCC50, logSI).
- Для классификации созданы бинарные метки:
    - Превышение медианы для IC50, CC50, SI
    - SI > 8.
- Признаки масштабированы с помощью StandardScaler (выше отмечено, что RobustScaler не дал сильного положительного эффекта).

## 4. Методология моделирования

Для задач регрессии обучены и сравнены несколько моделей:

    Ridge, RandomForest, GradientBoosting, XGBoost (были и другие, типа Lasso, но они не показали каких-то высоких результатов)
	
Для задач классификации обучены и сравнены несколько моделей:

    LogisticRegression, RandomForest, GradientBoosting, XGBoost.

Гиперпараметры подбирались с помощью GridSearchCV.

    - Метрики для регрессии: RMSE, MAE, R2.
    - Метрики для классификации: ROC-AUC, F1-score, precision, recall

Лучшие модели выбраны по R2 (регрессия) и ROC-AUC (классификация).

## 5. Разное

### 5.1. Влияние выбросов

- Удаление выбросов из целевых переменных не проводилось, поскольку они, вероятно, несут важную информацию.
- В классификации модели без удаления выбросов показали более высокое качество, чем с удалением выбросов.
- Для признаков с большим количеством выбросов использование RobustScaler не дало значительного прироста, поэтому в финальных моделях оставлено StandardScaler масштабирование.

### 5.2. Логарифмирование целевых переменных
- Логарифмирование целевых переменных существенно улучшило распределение и снизило влияние выбросов, что привело к более стабильным регрессионным моделям.
- Для классификации метки строились по исходным данным (без логифримирования).

### 5.3. Выбор моделей
- Ансамблевые методы (RandomForest, GradientBoosting, XGBoost) показывают более высокие результаты, чем линейные модели, за счёт способности улавливать нелинейные зависимости.
- Для регрессии CC50 лучшей оказалась RandomForest, для IC50 — GradientBoosting, для SI — также RandomForest.
- В классификации GradientBoosting и RandomForest показали сопоставимые результаты, причём для задачи SI > 8 лучшей стала RandomForest.

## 6. Заключение

В ходе работы протестированы модели для 7 задач прогнозирования для подбора наиболее эффективных сочетаний параметров для создания лекарственных препаратов. Наилучшие результаты достигнуты с помощью ансамблевых методов. Выявлено, что логарифмическая трансформация целевых переменных, незначительно, но улучшает качество регрессии, а удаление выбросов из данных нецелесообразно. Полученные модели могут быть использованы для отбора перспективных соединений при разработке лекарственных препаратов. 

В реальной жизни полагаться на результаты работы категорически нельзя!
